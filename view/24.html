Acceder a servicios con OAuth 2.0

Cree una aplicación que utilice credenciales de inicio de sesión de usuario designado para acceder a servicios seguros. (ver)
Dentro de codepen.io puede dar el siguiente problema en su ejecución:
Refused to display 'https://www.arcgis.com/sharing/oauth2/authorize?client_id=JTpyML5GgvA1jEoo&response_type=token&state=%7B%22portalUrl%22%3A%22https%3A%2F%2Fwww.arcgis.com%22%7D&expiration=20160&redirect_uri=https%3A%2F%2Fcdpn.io%2Fboomboom%2Fv2%2Findex.html%3Fkey%3DiFrameKey-84c13102-5496-0501-e371-156e9118149b' in a frame because it set 'X-Frame-Options' to 'sameorigin'.

Preconfigure OAuth 2.0 para su aplicación (esto son pasos previos para la ejecución de este ejercicio)

  1. Inicie sesión en su cuenta de ArcGIS. Si aún no tiene uno, regístrese gratis.
  2. En la parte superior derecha del menú principal, haga clic en + > Nueva aplicación > Crear nueva aplicación
    • Título: Access Premium Services App
    • Etiquetas: OAuth
    • Descripción: Access Premium Services App
    • Haga clic en Registrar nueva aplicación
  3. Haga clic en la pestaña Autenticación para agregar localhost o servername para el URI de redireccionamiento:
    • Redirigir URI http://localhost/
  4. Copie el valor de ID de cliente, ya que lo necesitará en el siguiente paso.
